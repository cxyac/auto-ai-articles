## 🎯 使用建议

### 1. 选择相关类别
根据当前任务类型，选择最合适的类别：
- 需要编写/调试代码 → Category 1
- 需要 AI 工作流设计 → Category 2
- 需要研究分析 → Category 3
- 需要自动化流程 → Category 4
- 需要创作内容 → Category 5
- 需要提高生产力 → Category 6

### 2. 每次使用 5 个提示词
不要一次性使用全部 300 个，而是：
- 挑选 5 个与当前任务最相关的
- 先用这 5 个
- 根据效果调整和优化
- 逐步熟悉所有 300 个

### 3. 填充模板
提示词中的 `[BRACKETS]` 部分是模板，需要你填充：
- `[DESCRIBE TASK]` → 描述你的具体任务
- `[DESCRIBE CONTEXT]` → 描述具体上下文
- `[DESCRIBE GOAL]` → 描述具体目标
- `[PASTE CODE]` → 粘贴你的代码
- `[DESCRIBE AUDIENCE]` → 描述目标受众

### 4. 持续优化
- 记录哪些提示词有效
- 记录哪些需要调整
- 逐步建立自己的提示词库
